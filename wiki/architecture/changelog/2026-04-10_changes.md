CHANGE-2026-04-10-001:
  Component: agent definitions
  What changed: Created Agent 14a (Assumption Extractor) and Agent 14b (Presumption Detector), splitting the original Agent 14. Files: wiki/agents/14a_assumption_extractor_agent.md, wiki/agents/14b_presumption_detector_agent.md.
  Why: "Finding presumptions is different than finding assumptions articulated already; maybe an agent 14a and 14b both feeding 15." [from Tom's review] Stated assumptions and unstated presumptions require different analytical approaches and carry different epistemic weight. The distinction enables the provenance/footnote protocol.
  Alternatives considered: Keep unified Agent 14 with both functions. Rejected because the two tasks require fundamentally different cognitive modes (extraction vs. inference).
  Downstream effects: Original Agent 14 definition superseded. Agent count rises. 14b runs after 14a to avoid duplication.
  Reversibility: Moderate (could re-merge, but provenance protocol depends on the split)
  Status: Implemented (definitions written; first run pending)

CHANGE-2026-04-10-002:
  Component: agent definitions
  What changed: Created Agent 15a (Literature Search FOR) and Agent 15b (Literature Search AGAINST). Files: wiki/agents/15a_lit_search_for_agent.md, wiki/agents/15b_lit_search_against_agent.md.
  Why: "Aren't there different functions in 15 as well? So 15a, b..." [from Tom's review] Searching FOR and searching AGAINST must be independent to prevent confirmation bias. Neither agent sees the other's results during a search cycle.
  Alternatives considered: Single Agent 15 doing both searches. Rejected because a single agent searching both directions would inevitably be biased by whichever direction it searches first.
  Downstream effects: Closes the empirical grounding loop (14a/14b → 15a/15b → 14a/14b). Adds 2 agents. Requires for_lit_search.md intake queue and lit_search_results/ directories.
  Reversibility: Moderate
  Status: Implemented (definitions written; first run pending)
