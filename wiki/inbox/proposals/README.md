# Proposal System — How It Works

Proposals are candidate additions to the Wiki, generated automatically by the hunt phase of the daily scheduled task. They are **never ingested directly** — each one waits for Tom's review.

---

## Lifecycle

```
Hunt phase finds new paper/talk/interview
        ↓
Proposal written to: proposals/pending/YYYY-MM-DD_[tradition]_[slug].md
        ↓
Daily review page generated: wiki/review/YYYY-MM-DD_review.html
        ↓
Email sent to thomas.loughran@gmail.com with links
        ↓
Tom reviews each proposal (Approve / Double-check / Change / Deny)
        ↓
Tom sends decision email (generated by review page "Submit" button)
        ↓
Next morning: scheduled task reads decision email via Gmail
        ↓
APPROVE → proposal moved to wiki/inbox/ → ingested by agents next run
DENY    → moved to proposals/denied/
CHECK   → moved to proposals/needs_review/ with note
CHANGE  → Tom's edit note appended; moved to proposals/needs_review/
        ↓
Decision written to: wiki/review/archive/YYYY-MM-DD_decisions.md
Review page deleted from wiki/review/
```

---

## Proposal File Format

```markdown
---
proposal_id: PROP-YYYY-MM-DD-NNN
thinker: [Full Name]
tradition_key: [folder key, e.g. friston]
source_type: [paper | talk | interview | blog | podcast]
source_title: "[exact title]"
source_url: [URL]
source_date: [YYYY-MM-DD or approx]
searched_on: [YYYY-MM-DD]
status: [pending | approved | denied | needs_review]
---

## Summary
[2-3 sentence plain-language summary of the source]

## Why This Matters for This Tradition
[1-2 sentences on relevance to the thinker's research program]

## Candidate PRS Triplets

PRS-CANDIDATE-01:
  Problem: [what unsolved question this addresses]
  Resource: [concept, model, finding, or tool introduced]
  Solution: [what advance is proposed or demonstrated]
  Confidence: [High / Medium / Speculative]
  Evidence: [specific claim or finding in the source that supports this]

[add more if warranted — max 3 per proposal]

## Cross-Tradition Signals
[optional — any connections to other programs noticed during hunting]
```

---

## Folders

| Folder | Contents |
|--------|----------|
| `pending/` | New proposals awaiting review |
| `approved/` | Approved — moved to wiki/inbox/ for ingestion |
| `denied/` | Declined — kept for reference, not ingested |
| `needs_review/` | Flagged for double-check or change — returned to pending after resolution |

---

## Decision Email Format

The review page generates this automatically. Do not edit — the task parser depends on exact formatting.

```
Subject: [C2A2-review-decision] YYYY-MM-DD

DECISIONS:
PROP-YYYY-MM-DD-001: APPROVE
PROP-YYYY-MM-DD-002: DENY
PROP-YYYY-MM-DD-003: CHECK | [optional note]
PROP-YYYY-MM-DD-004: CHANGE | [specific change instruction]
```
